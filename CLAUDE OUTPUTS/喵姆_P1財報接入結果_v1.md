## 修改內容

### 修改檔案：`scripts/stock_updater.py`

#### 新增 1：FinMind 財報 type name 映射（class 層級常數）

```python
_INCOME_TYPE_MAP = {
    "revenue": ["Revenue", "營業收入", "營業收入合計", "收入"],
    "net_income": ["NetIncome", "本期淨利（淨損）", "本期綜合損益總額",
                   "歸屬於母公司業主之淨利（損）", "淨利（淨損）", "稅後淨利"],
    "non_operating_income": ["NonOperatingIncome", "營業外收入及支出合計",
                              "營業外收入及支出", "營業外損益"],
    "pretax_income": ["PreTaxIncome", "稅前淨利（淨損）", "繼續營業單位稅前淨利（淨損）",
                      "稅前淨利", "繼續營業單位稅前損益"],
}
_BALANCE_TYPE_MAP = {
    "accounts_receivable": ["AccountsReceivable", "應收帳款淨額", "應收帳款",
                            "應收帳款及票據", "應收帳款－淨額"],
    "inventory": ["Inventory", "存貨", "存貨淨額", "存貨合計"],
}
_CASHFLOW_TYPE_MAP = {
    "operating_cash_flow": ["OperatingCashFlow", "營業活動之淨現金流入（流出）",
                            "營業活動之現金流量", "營業活動淨現金流入（流出）",
                            "營業活動之現金流量－合計"],
}
```

**設計決策：** 因為 FinMind 的 `type` 欄位可能隨版本/公司不同而有不同中文名稱（例如「本期淨利（淨損）」vs「歸屬於母公司業主之淨利（損）」），每個引擎欄位都配了多個候選名，第一個匹配到的就用。

#### 新增 2：`_fetch_finmind_dataset()` — 通用 FinMind 抓取函數

複用現有的 `self.finmind_api` 和 `self.finmind_token`，支援任意 dataset。

#### 新增 3：`_resolve_type_value()` — 從 long-format 抽取值

FinMind 回傳的是 long-format（每列一個指標），這個函數根據 type_map 把它轉成 engine 需要的 dict。

#### 新增 4：`fetch_financial_data()` — 主要財報抓取函數

```
流程：
1. 呼叫 TaiwanStockFinancialStatements（損益表）
2. 呼叫 TaiwanStockBalanceSheet（資產負債表）
3. 呼叫 TaiwanStockCashFlowsStatement（現金流量表）
4. 找出最近兩季的日期
5. 對每季用 _resolve_type_value() 映射到 engine 欄位
6. 回傳 {"quarters": [{latest}, {prev}]}
```

附帶 **debug logging**：首次呼叫時會列出每張報表的所有 type names，方便未來調整映射。

#### 修改 5：`build_engine_input()` — 新增 `financial_data` 參數

```python
# 修改前
def build_engine_input(self, code, history, chip_data):
    return {"financial": {}, ...}

# 修改後
def build_engine_input(self, code, history, chip_data, financial_data=None):
    return {"financial": financial_data if financial_data else {}, ...}
```

#### 修改 6：`update_single()` — 加入 fetch_financial_data 呼叫

```python
# 在抓籌碼和組裝 engine input 之間新增：
financial_data = self.fetch_financial_data(code)
engine_input = self.build_engine_input(code, history, chip_data, financial_data)
```

每檔股票多 3 個 API 呼叫（損益+資產負債+現金流量），加上 0.3 秒間隔 rate limit。

### 關鍵突破

- **BUY 判定首次觸發！** P0 後最多 2/4 base bullish（不夠 ≥3），P1 加入財報後達到 3/4 → BUY 條件達成。
- **修復前：** 4/4 全部 WAIT/Low（零鑑別力）
- **修復後：** 3 種裁決（BUY/MEDIUM + SELL/HIGH + WAIT/LOW），鑑別力完全恢復
- **Confidence 梯度：** BUY=MEDIUM（因 sentiment 仍 neutral），SELL=HIGH（3 個 base 均 bearish/danger）

### 為何 BUY 是 MEDIUM 而非 HIGH？

VerdictEngineV2 的 BUY confidence 計算依賴多個因素，其中 market_sentiment 仍為 neutral fallback。接入情緒資料（P2）後，全方位多頭股票有望達到 BUY/HIGH。

## 已知限制與風險

### 1. FinMind type name 映射可能不完整

已對每個欄位提供 4-5 個候選名稱，但某些特殊公司（金控、KY 股）的報表科目名稱可能不同。

**減緩措施：** `fetch_financial_data()` 會 log 所有 type names，首次真實跑時觀察 log 即可發現缺漏。

### 2. 財報資料有季度延遲

台股季報公布時間：Q1→5月中、Q2→8月中、Q3→11月中、年報→3月底。因此「最新季」可能是 2-3 個月前的資料。

**影響：** FinancialForensicsV2 分析的是公司體質（較穩定），2-3 個月延遲可接受。

### 3. 資產負債表/現金流量表日期可能與損益表不完全對齊

不同報表的公布日期可能差幾天。程式碼已用「最接近日期」邏輯處理。

### 4. 每檔股票多 3 個 API 呼叫

33 檔更新時間從約 2 分鐘增加到約 3-4 分鐘（多了 99 個 API 呼叫 + rate limit 間隔）。
