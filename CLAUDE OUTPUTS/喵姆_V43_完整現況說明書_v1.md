## 1. 專案概述

### 1.1 項目定義
喵姆看盤是一個**台灣股票 AI 分析系統**，專為個人投資者設計。它結合了四大 AI 模型（Gemini Flash、Claude 3.5、Perplexity、GPT-4o）的協同分析能力與台灣市場深度資料（三大法人、融資融券、月營收），為用戶提供每日自動化的投資決策建議。

### 1.2 核心定位
- **非**通用看盤軟體（如 TradingView、Bloomberg Terminal）
- **是**個人 AI 投資分析師助理，強調「自動化」與「AI 協同」
- **適合**「下班後查看分析報告」的策略型投資者
- **不適合**盤中即時交易者（無盤中推送、延遲化設計）

### 1.3 技術棧
| 項目 | 內容 |
|-----|------|
| **前端** | 單頁應用 (SPA)，純 HTML/CSS/JS，無框架 |
| **圖表庫** | LightweightCharts 4.1.0（蠟燭圖、量能、指標疊加） |
| **AI 模型** | Gemini Flash、Claude 3.5、Perplexity、GPT-4o（四同時） |
| **後端語言** | Python 3（資料抓取、分析計算、排程） |
| **資料源** | FinMind API、TWSE、Google Sheets、自有資料庫 |
| **雲端同步** | Google Apps Script + Google Sheets |
| **部署** | GitHub Pages（靜態）+ Python 後端（定時排程） |
| **PWA 支援** | Service Worker + Manifest，可離線基本使用 |

### 1.4 檔案規模
| 檔案 | 大小 | 說明 |
|-----|------|------|
| index.html（主程式） | ~374 KB | CSS + HTML + JS 全在一個檔案 |
| daily_analysis.json | ~512 KB | 全部追蹤股票的分析資料 |
| daily_analysis.json | ~14 KB | 大盤歷史資料 |
| 個股 K 線資料 | ~36 檔 | 各股 data/*.json |

#### 2.3.2 資料流程

```
14:30 台灣股市收盤
  ↓
v35_runner.py 啟動
  ├── ① stock_updater.py
  │   ├─ FinMind API → 獲取股價、成交量
  │   ├─ 計算技術指標 (MA/RSI/MACD/Bollinger)
  │   ├─ miaomu_engine_v2 → 判定 BUY/SELL/WAIT
  │   └─ 寫回 daily_analysis.json
  │
  ├── ② index_monitor.py
  │   ├─ TWSE API → 大盤資料
  │   └─ 寫入 market_history.json
  │
  ├── ③ chips_monitor.py
  │   ├─ 三大法人買賣超
  │   └─ 融資融券資訊
  │
  ├── ④ line_alerts.py
  │   └─ 推播到 LINE
  │
  └─ git add → git push → GitHub Pages
      ↓
前台 fetch daily_analysis.json（每次打開或手動刷新）
  ↓
renderVerdict() 顯示今日分析結果
```

## 6. 已知限制與待改善項目

### 6.1 技術分析維度的限制

| 限制 | 影響 | 優先級 |
|-----|------|--------|
| 無繪圖工具 | 用戶不能手繪趨勢線、通道 | 🔴 |
| 指標庫有限 | 缺少 Ichimoku、Stochastic、DMI 等 | 🔴 |
| 無週線/月線完整 K | 只聚合計算，不是真正的週線圖 | 🔴 |
| 無多圖同屏 | 不能看 2-4 檔股票同步對比 | 🟡 |
| 無全屏模式 | K 線圖被側邊欄壓縮 | 🟡 |

### 6.2 基本面資料的限制

| 限制 | 影響 | 優先級 |
|-----|------|--------|
| 無財報深度 | 只有單行數字，無 5 年曲線 | 🔴 |
| 無毛利率趨勢 | 無法看長期成長性 | 🔴 |
| 無同業對標 | 看不到相對位置 | 🔴 |
| 無 PEG 評分 | 無法判斷成長股估值 | 🟡 |
| 分析師資訊延遲 | 來源不清、更新頻率未知 | 🟡 |

### 6.3 AI 能力的限制

| 限制 | 影響 | 優先級 |
|-----|------|--------|
| 無訓練履歷 | 無法看模型在此股的歷史準確率 | 🟡 |
| 無機器學習 | 固定規則，無自適應 | 🟡 |
| 無因子模型 | 無法追蹤 10+ 因子相關性動態變化 | 🟢 |
| 無組合優化 | 倉位配置無 Markowitz 優化 | 🟢 |

### 6.4 即時性的限制

| 限制 | 影響 | 優先級 |
|-----|------|--------|
| 無盤中 K 線更新 | 只能看昨日收盤線 | 🟡 |
| 無盤中推送 | 訊號要等使用者主動查看 | 🟡 |
| 無 WebSocket 推送 | 基於 API 輪詢，延遲 1+ 分鐘 | 🟡 |
| 無選擇權報價 | 台灣選擇權未整合 | 🟢 |
| 無期貨 tick 級 | 期貨資料可能延遲 15 分鐘 | 🟢 |

## 12. 用戶指南概要

### 12.1 日常操作流程

```
每個交易日的典型使用流程：

下午 14:30 後（盤後）
  ├─ 打開喵姆看盤（可能資料已自動更新）
  ├─ 首頁查看追蹤股票的今日判定
  ├─ 點進關鍵股票檢視完整分析
  └─ 後台「我的持股」確認 P&L + 風控警示

傍晚（設定警報或下單前）
  ├─ 後台「風控中心」查看 AI 委員會意見
  ├─ 調整停損/目標價位
  └─ 根據分析執行交易

隔日開盤後
  ├─ 後台「每日訊號」掃描強勢/弱勢股
  ├─ 選擇性加碼看多或減碼看空的股票
  └─ 持續追蹤，等下午收盤重複流程
```

1. `daily_analysis.json` 的 `generated_at` 是今天嗎？

```bash
export $(grep -v '^#' .env | xargs)
```
