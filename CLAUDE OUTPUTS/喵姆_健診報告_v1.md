## 面向 3：評分可信度評估

### 7 維度真實性一覽

| 維度 | 資料來源 | 真實性 | Score 分布 | Signal |
|------|----------|--------|------------|--------|
| 籌碼結構 | FinMind 法人買賣超 | ✅ 真實 | 3.5–9.0（9種不同值） | bullish / neutral |
| 技術面診斷 | FinMind 價格→技術指標 | ✅ 真實 | 2.5–6.0（4種不同值） | bearish / neutral |
| 惡魔代言人 | 衍生自其他維度 | ⚠️ 半真實 | 全部 6.0 | 全部 neutral |
| 財報真實性 | 空 dict fallback | ❌ 假資料 | 全部 5.0 | 全部 neutral |
| 市場情緒 | 空 dict fallback | ❌ 假資料 | 全部 5.0 | 全部 neutral |
| 產業結構 | 空 dict fallback | ❌ 假資料 | **全部 2.0** | **全部 danger** |
| 先行指標 | 空 dict fallback | ❌ 假資料 | 全部 5.0 | 全部 neutral |

### 關鍵數字

- **真實資料維度：2/7（29%）** — 只有籌碼結構和技術面是從真實市場數據計算
- **半真實維度：1/7（14%）** — 惡魔代言人依賴其他維度推導，但因大部分輸入是假的，輸出也失真
- **假資料維度：4/7（57%）** — 超過一半維度使用引擎預設 fallback

### 🚨 最嚴重問題：產業結構 Fallback 系統性偏差

**問題核心：** `IndustryStructure({})` fallback 回傳 score=2.0、signal=danger

**連鎖影響：**
1. VerdictEngineV2.judge() 中有一條規則：`if struct_dim.score < 4 → conf = Confidence.LOW`
2. 因為所有股票的 struct_dim.score = 2.0（< 4），**所有 BUY 判定都被強制降為 LOW confidence**
3. 加上 danger signal 計入 bearish 陣營，拉低整體分數

**結果：33/33 支股票全部判定「觀望」（wait），0 支買進、0 支賣出**

**最極端案例：**
- 廣達（2382）：籌碼結構 score=9.0（bullish），技術面 score=6.0（neutral）
  → 理應是強烈買進訊號
  → 但產業結構 2.0/danger 拖累 → 最終 verdict=wait, confidence=LOW

### 總評分分布

|------|-----|
| 最高分 | 5.4（廣達、長榮、鈊象） |

**可信度評級：🔴 低 — 系統性假資料汙染導致所有判定失去鑑別力**

## 面向 4：allorigins.win 風險確認

### 使用位置

| 行號 | 功能 | URL 樣式 |
|------|------|----------|
| 5759-5760 | `corsProxy()` 函數定義 | `https://api.allorigins.win/raw?url=` |
| ~5790 | 三大法人（BFI82U） | TWSE API → allorigins proxy |
| ~5802 | 融資融券（MI_MARGN） | TWSE API → allorigins proxy |
| ~5917 | Yahoo Finance RSS | Yahoo → allorigins proxy |

### 風險評估

| 風險 | 等級 | 說明 |
|------|------|------|
| 服務中斷 | 🔴 高 | allorigins.win 是免費第三方服務，無 SLA，隨時可能停機或限流 |
| 資料竄改 | 🟡 中 | 中間人可以修改回傳資料（但 HTTPS 有一定保護） |
| 隱私洩漏 | 🟡 中 | 請求 URL 含股票代號，allorigins 可以看到使用者在查什麼 |
| 速率限制 | 🟡 中 | 免費服務通常有請求上限，多人使用可能被 ban |
| 單點失敗 | 🔴 高 | 3 個即時資料源全靠同一個 proxy，掛了全斷 |

### 不受影響的部分

- 大盤指數（mis.twse.com.tw）直連，不經過 allorigins ✅
- 後端 stock_updater.py 使用 FinMind API 直連，不經過 allorigins ✅
