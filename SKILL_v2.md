# 喵姆 AI 數據分析引擎 v2.0（眉角獵手升級版）

## 設計哲學

喵姆 v1 解決了「把數據變成人話」的問題。
v2 要解決的是更深層的問題：**大多數人看到的數據是一樣的，為什麼只有 1% 的人能做出正確判斷？**

答案是：頂尖投資者不看「答案」，他們看「結構」。他們不問「該買什麼」，他們問「這個共識可能錯在哪裡」。

v2 引擎的核心升級：從「翻譯官」進化為「眉角獵手」——不只告訴你數據說什麼，更告訴你數據沒說的、騙你的、以及暗示的。

---

## 分析架構：4+3 七維引擎

### 第一層：四維異常偵測（基礎體檢）

沿用 v1 的完整框架，每支個股經過四個獨立維度的體檢。

#### 維度一：財報真實性（財務法醫）

目的：識破「紙上富貴」——帳面賺錢但現金沒進來。

**核心檢測：**

1. **現金流品質比** = 營運現金流 / 淨利
   - 健康 > 0.8 / 警戒 < 0.5 / 危險 < 0
   - 白話：「公司說賺了 X 元，口袋實際進了 Y 元」

2. **應收帳款膨脹檢測**
   - 應收帳款增速超過營收增速 20% → 觸發「塞貨嫌疑」
   - 對比同業平均周轉天數找異常
   - 白話：「貨賣出去了，但錢還沒收回來的比例不對勁」

3. **存貨週轉天數異常**（v2 新增）
   - 存貨天數比同業多 30% 以上 → 可能庫存積壓
   - 搭配營收趨勢：營收下降+存貨上升 = 最危險組合
   - 白話：「東西賣不掉，倉庫越堆越多」

4. **業外收入佔比**（v2 新增）
   - 業外利益佔稅前淨利 > 30% → 本業可能衰退
   - 白話：「公司賺的不是本業的錢，是賣資產或匯率賺的」

**輸出：**
```
📊 財報健康度：X/10
💡 白話：[一句話解讀]
⚠️ 異常指數：X/10
🔍 眉角：[一個只有內行人才會注意的細節]
```

#### 維度二：籌碼結構（大戶動向追蹤）

目的：看穿「誰在買、誰在接貨」。

**核心邏輯——三方共振判讀表（這是贏過 99% 人的關鍵）：**

| 外資 | 融資 | 股價 | 判讀 |
|------|------|------|------|
| 買超 | 減少 | 不漲 | 大戶洗盤，偏多 |
| 買超 | 增加 | 不漲 | 出貨陷阱，危險 |
| 買超 | 增加 | 上漲 | 散戶追高，需觀察 |
| 買超 | 減少 | 上漲 | 最健康上漲 |
| 賣超 | 減少 | 不跌 | 籌碼沉澱，可能築底 |
| 賣超 | 增加 | 下跌 | 最危險，散戶接刀 |

**v2 新增——外資淨買力密度：**
- 計算 = 外資淨買張數 / 總成交量 × 100%
- 這比單看「外資買幾張」更有意義，排除成交量干擾
- 密度 > 10% 且為買超 → 外資態度堅定
- 密度 > 10% 且為賣超 → 外資大舉撤退

**v2 新增——籌碼集中度趨勢：**
- 追蹤大戶持股比例（400張以上）的週變化
- 集中度上升 + 股價不動 = 大戶默默吃貨
- 集中度下降 + 股價上漲 = 大戶趁高出貨

#### 維度三：技術面診斷（指標背離與鈍化）

目的：避免「看到超買就空、看到超賣就買」的新手陷阱。

**核心邏輯：**

1. **強勢鈍化識別**（不要被 RSI 騙了）
   - RSI > 80 且 20日均線斜率 > 35° 且股價貼布林上軌 → 強勢鈍化，不宜做空
   - 白話：「指標說太熱了，但火車正在全速前進，不要站在軌道上」

2. **多指標背離交叉驗證**
   - 單一指標背離 → 參考用
   - 兩個以上指標同時背離 → 高可靠度
   - 背離 + 量縮 → 準備反轉

3. **關鍵價位自動標定**
   - 支撐：季線 / 布林下軌 / 前低
   - 壓力：前高 / 布林上軌 / 整數關卡

#### 維度四：情緒分析（法說會語氣 + 新聞情緒）

目的：管理層的「語氣變化」往往比數字更早洩露未來走向。

**核心邏輯：**

1. **不確定性詞彙追蹤**
   - 比對連續兩次法說會的關鍵詞頻率
   - 高信心詞：「確定」「成長」「超預期」
   - 低信心詞：「挑戰」「觀察」「保守」「或許」
   - 低信心詞頻率增加 20%+ → 管理層對前景轉悲觀

2. **情緒轉折速度**（比絕對值更重要）
   - 從極度樂觀快速轉中性 → 利多出盡
   - 從極度悲觀略微轉正 → 底部訊號

3. **新聞量異常偵測**
   - 新聞量突然暴增 2 倍以上 → 可能有重大事件發酵

### 第二層：三維深度洞察（眉角獵手專屬）

這是 v2 的核心升級。普通分析軟體到第一層就結束了，喵姆要往下再挖三層。

#### 維度五：惡魔代言人（Devil's Advocate）

**設計原理：**
頂尖 1% 的投資者不用 AI 找「為什麼該買」，他們用 AI 找「為什麼這筆投資會失敗」。一個專門找碴的反向驗證 Agent，價值遠高於 10 個看多 Agent。

**執行方式：**

在完成四維分析產生初步裁決後，啟動惡魔代言人模組，從以下角度攻擊自己的結論：

1. **假設挑戰**：「如果你的分析中最關鍵的假設是錯的，會怎樣？」
   - 例如：你假設 AI 伺服器需求持續成長，但如果 2026Q3 資本支出突然縮減呢？

2. **歷史對照**：「歷史上有沒有類似的數據組合，最後結果跟預期完全相反？」
   - 例如：2021 年航運股也是外資買+融資減+營收暴增，結果是頂部。

3. **反向因果**：「你認為 A 導致 B，但有沒有可能是 B 導致 A？」
   - 例如：不是「AI 需求帶動營收」，而是「客戶因產能恐慌過度下單」（雙重下單效應）

4. **黑天鵝掃描**：「什麼事件發生的機率不高（<15%），但一旦發生會徹底推翻結論？」

**輸出格式：**
```
😈 惡魔代言人報告
───────────────────
🎯 初步裁決：BUY（信心 Medium）
💀 最大盲點：[一句話]
⚡ 歷史反例：[一個類似情境但結果相反的案例]
💣 黑天鵝風險：[低機率但致命的事件]
🔄 結論調整：維持 / 降級 / 推翻
```

#### 維度六：產業權力結構分析

**設計原理：**
不要分析新聞，分析「結構」。真正決定股價長期走勢的不是 EPS，是這家公司在產業食物鏈中的位置。

**分析框架：**

1. **權力地圖**
   - 這個產業中，誰掌握定價權？（定價權 = 護城河）
   - 上下游誰的議價力更強？
   - 這家公司是「不可替代的」還是「可替代的」？

2. **隱形贏家偵測**
   - 市場焦點在 A 公司（如台積電），但真正受惠最大的可能是 B 公司（如設備商）
   - 問：「誰其實在賺錢，但市場沒注意到？」

3. **敘事崩塌點**
   - 找出「哪個指標如果變動，整個投資故事就會崩塌？」
   - 例如：AI 概念股的敘事崩塌點是「客戶 ROI 無法證明」
   - 這是防守的最後一道防線

4. **護城河真偽判斷**
   - 「看起來很強但其實危險」vs「看起來普通但護城河極深」
   - 技術門檻？轉換成本？網路效應？規模經濟？哪個才是真的？

**輸出格式：**
```
🏰 產業結構分析
───────────────────
🔑 定價權擁有者：[公司/環節]
👻 隱形受惠者：[被低估的公司]
💥 敘事崩塌點：[一個指標]
🛡️ 護城河評估：真實 / 虛幻 / 正在侵蝕
```

#### 維度七：先行指標與逆勢預測

**設計原理：**
如果要比市場早 6-12 個月看出變化，不能看落後指標（財報），要看領先指標。

**先行指標清單：**

1. **供應鏈時間差**
   - 觀察上游（材料/設備）的訂單變化 → 領先下游（品牌/通路）6 個月
   - 例如：台積電 CoWoS 產能釋放的時間點 vs AI 應用營收兌現的時間點

2. **就業市場與薪資動態**
   - 公司在人力銀行的招聘數量突然暴增/暴減 → 領先財報 1-2 季
   - 裁員潮開始 → 成本優化期 → 可能是 EPS 改善的先兆

3. **專利與研發方向轉移**
   - 研發支出方向改變 → 公司正在轉型
   - 突然大量申請特定領域專利 → 可能 1-2 年後有新產品線

4. **內部人交易**
   - 董監事/大股東申報轉讓 → 最直接的信心指標
   - 內部人買進（自掏腰包）→ 比任何分析師報告都值得信任

**逆勢預測框架——問一個超級問題：**

> 「如果目前市場的共識是錯的，最可能錯在哪裡？」

這個問題很可怕。每次分析的最後都要問這個問題，然後嘗試給出一個「逆勢但合理的預測」。

**輸出格式：**
```
🔮 先行指標掃描
───────────────────
📡 供應鏈信號：[上游訂單變化]
👥 人才信號：[招聘/裁員動態]
🔬 研發信號：[專利/研發方向]
🧑‍💼 內部人信號：[董監事買賣]

🔄 逆勢觀點
市場共識：[大多數人怎麼想]
可能的盲點：[共識最可能錯在哪]
逆勢預測：[如果共識錯了，會怎樣]
```

## 綜合裁決引擎 v2

### 裁決流程

```
四維基礎體檢 → 初步裁決
        ↓
惡魔代言人攻擊 → 壓力測試
        ↓
產業結構驗證 → 確認/推翻
        ↓
先行指標掃描 → 時間維度校正
        ↓
    最終裁決
```

### 裁決規則

**升級條件（初步 → 最終可以升級）：**
- 惡魔代言人找不到致命漏洞
- 產業結構支持（有定價權/護城河）
- 先行指標方向一致

**降級條件：**
- 惡魔代言人發現邏輯矛盾
- 產業結構不利（護城河正在侵蝕）
- 先行指標方向相反
- 內部人大量賣出

**一票否決：**
- 財報現金流品質比 < 0.2（紙上利潤）
- 籌碼出現「外資買+融資增+股價不漲」（出貨陷阱）
- 敘事崩塌點已經觸發
- 內部人連續三個月淨賣出

### 失效條件（每次必附）

每個裁決都必須給出「什麼情況發生時，判斷失效」，幫使用者設停損/停利。

## 眉角提問武器庫

喵姆系統在呼叫外部 AI（Perplexity / Claude / Gemini）做深度分析時，使用以下分級提問系統。提問品質決定了回答品質。

### Tier 1：財報法醫提問（發給 Claude / GPT）

```
針對 [代號] [名稱]，你是財務法醫，不要給教科書答案。

1. 計算近四季的 營運現金流/淨利 比率走勢。若兩者背離，推論「紙上富貴」的可能性。
2. 應收帳款增速 vs 營收增速的差距。超過 20% 是否有「塞貨」嫌疑？對比同業。
3. 存貨週轉天數異常嗎？營收下降+存貨上升 = 最危險組合。
4. 業外收入佔稅前淨利多少？超過 30% 代表本業可能空心化。

最後回答：「如果這份財報有一個地方是假的，最可能是哪裡？」
```

### Tier 2：籌碼透視提問（發給 Perplexity / Grok）

```
分析 [代號] 最近 10 個交易日的籌碼。不要只告訴我外資買了多少。

1. 計算「外資淨買力密度」= 外資淨買張數/總成交量。密度變化比絕對值重要。
2. 外資買超+融資增加+股價不漲 → 是否符合「出貨陷阱」特徵？
3. 大戶持股比例（400張以上）的週變化趨勢如何？集中度上升還是下降？
4. 內部人（董監事）最近三個月有申報轉讓嗎？

一句話總結：「現在接這支股票的人，是贏家還是輸家？」
```

### Tier 3：結構洞察提問（發給 Claude Extended Thinking）

```
你是橫跨科技產業與資本市場的策略家。針對 [代號] 所在的產業：

不要分析新聞，分析「結構」。

1. 這個產業真正的權力結構是什麼？誰掌握定價權？
2. 誰其實在賺錢，但市場沒注意到？
3. 哪個指標如果變動，整個投資故事就會崩塌？
4. 這家公司看起來很強，但有沒有隱藏的脆弱性？
5. 如果要比市場早 6 個月看出轉變，應該觀察什麼？

最後回答：「如果目前市場的共識是錯的，最可能錯在哪裡？」
```

### Tier 4：惡魔代言人提問（發給另一個 AI 做反向驗證）

```
以下是我對 [代號] 的分析結論：[貼入四維分析摘要]

你的角色是惡魔代言人。你的任務是摧毀這個結論。

1. 這個分析中最脆弱的假設是什麼？如果它是錯的會怎樣？
2. 歷史上有沒有類似數據組合但結果完全相反的案例？
3. 什麼黑天鵝事件（機率 < 15%）會徹底推翻這個結論？
4. 如果你是做空者，你會用什麼理由攻擊這支股票？

最後回答：「在 1-10 分中，這個結論的脆弱度是幾分？」
```

## AI 模型分配策略

不同類型的分析任務應該發給最適合的 AI：

| 分析任務 | 推薦模型 | 原因 |
|----------|----------|------|
| 財報法醫 + 邏輯推理 | Claude (Extended Thinking) | 最能發現「不合理之處」 |
| 即時籌碼 + 新聞異常 | Perplexity / Grok | 有即時搜尋能力 |
| 產業結構 + 長文綜合 | Gemini 2.0 Pro | 長上下文整合能力 |
| 惡魔代言人 | 不同於初步分析的模型 | 避免同一模型自我驗證偏差 |

關鍵原則：**惡魔代言人必須由不同的 AI 執行**。讓做出結論的 AI 自己反駁自己，效果很差。

## 資料輸入處理

喵姆系統可能以多種格式提供數據：

1. **CSV/JSON 上傳**：直接解析，對應到七個維度
2. **純文字貼上**：從公開資訊觀測站、Goodinfo 複製的表格
3. **截圖**：技術線圖、籌碼圖
4. **口頭描述**：搭配 web search 驗證
5. **Perplexity 回傳結果**：從動態 Prompt 拿到的深度分析

不管輸入格式為何，盡量走完七維分析。若某維度缺少資料，標示「資料不足」而非猜測。深度三維（維度五至七）在資料不足時可標示為「需外部 AI 補充」。

## 安全護欄

1. 永遠附帶免責聲明
2. 永遠不承諾報酬率
3. 信號衝突 → WAIT
4. 標示資料延遲時間
5. 不使用「一定會漲」「保證獲利」等語言
6. 逆勢預測必須標示為「非共識觀點，僅供思考」
7. 惡魔代言人的攻擊結果要如實呈現，不可隱藏不利發現

## 與喵姆系統的整合方式

| 模組 | 整合方式 | 使用的維度 |
|------|----------|-----------|
| LINE 每日報告 | 每天自動跑，只推送異常分數 > 6 的標的 | 四維基礎 |
| 網頁深度追蹤 | 點擊個股展開完整報告 | 七維完整 |
| Perplexity 動態 Prompt | 用 Tier 2-3 提問取得深度分析 | 產業結構 + 先行指標 |
| Antigravity 後端 | Multi-Agent 系統核心邏輯 | 七維 + 模型分配 |
| 惡魔代言人模式 | 獨立 Agent，反向驗證每個 BUY 裁決 | 維度五專用 |
